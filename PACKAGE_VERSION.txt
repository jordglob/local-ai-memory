Package release versioning
==========================
The zip filename carries a package-level release number + author + date so the
newest, authoritative bundle is always obvious and confusions (two near-identical
zips) can't happen.

Scheme:  local-ai-memory-repo_v<N>_<author>_<YYYY-MM-DD>.zip
  <N>       running package release number, +1 each consolidated zip
  <author>  who produced it: "WEB" (Claude in chat), "CC" (Claude Code). Nothing
            machine-specific — the tool is generic, not tied to any one box.
  <date>    build date

This is SEPARATE from the per-script versions (setup v8.6, ingest v2.4, spec
v1.10 ...) which move independently. The package number just answers
"which bundle is newest?".
